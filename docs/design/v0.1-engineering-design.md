# AI Cloud v0.1 Engineering Design

## 1. Goal

Build a runnable AI Cloud MVP based on:

- AI Control Plane
- Token Gateway Data Plane
- Agent Execution Plane

The first scenario is Developer AI Cloud: a secure coding agent platform.

## 2. Architecture Boundary

```
Client / IDE / CLI
        |
        v
AI Cloud API
        |
Control Plane
        |
Gateway Plane
        |
Agent Execution Plane
        |
Models + Enterprise Tools
```

## 3. Services

### control-plane
Responsible for:

- tenant
- identity
- task lifecycle
- model selection
- policy decision

### model-service
Responsible for:

- model registry
- provider adapter
- capability metadata
- pricing metadata

### agent-runtime
Responsible for:

- agent lifecycle
- planning
- execution context
- tool invocation

### workflow-service

Temporal based long running workflow engine.

### tool-gateway

Enterprise tool access boundary.

### policy-service

OPA/Rego based authorization and safety control.

## 4. Core Objects

```
Model

```

## 5. MVP Execution Flow

```
Create Task
   |
Select Agent
   |
Select Model
   |
Create Sandbox
   |
Execute Tools
   |
Validate Result
   |
Record Trace and Cost
```

## 6. Technology Decision

|Area|Choice|
|-|-|
|Backend|Go|
|Database|PostgreSQL|
|Cache|Redis|
|Workflow|Temporal|
|Gateway|LiteLLM initially|
|Policy|OPA|
|Runtime|Kubernetes|
|Observability|OpenTelemetry|

## 7. Non Goals

Not building:

- foundation model training
- GPU scheduler
- generic chatbot
- replacement of existing gateways

AI Cloud focuses on enterprise AI orchestration and governance.
