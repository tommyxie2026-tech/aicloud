Current implementation boundary:


This remains a skeleton implementation and is not a general YAML parser.

## Desired PR-038 Behavior

PR-038 should change `DryRunManifestWriter` from object-only patch output toward byte-level manifest output.

Intended high-level flow:

```text
ManifestPatchPlan
  + source ManagedCluster YAML bytes
  ↓
yamlio.ReadManagedCluster
  ↓
ApplyManagedClusterPatch
  ↓
yamlio.WriteManagedCluster
  ↓
WriteResult with updated object and rendered manifest bytes or planned file content
```

## Recommended Minimal Interface Change

Avoid giving `DryRunManifestWriter` direct filesystem responsibility.

Prefer adding a method that accepts bytes:

```go
WriteManagedClusterBytes(plan ManifestPatchPlan, source []byte) (*WriteResult, error)
```

Keep the existing object-level method for tests and internal composition:

```go
WriteManagedCluster(plan ManifestPatchPlan, cluster api.ManagedCluster) (*WriteResult, error)
```

## WriteResult Extension Candidate

If needed, extend `WriteResult` with rendered content metadata:

```go
Rendered []byte
```

or a more explicit field name:

```go
RenderedManifest []byte
```

Do not introduce filesystem paths beyond the existing `SourcePath` and `OutputPath` planning fields.
