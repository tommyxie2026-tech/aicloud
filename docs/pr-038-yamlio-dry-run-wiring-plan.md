# PR-038 yamlio DryRunManifestWriter Wiring Plan

## Goal

Prepare the next step after PR-037 stabilization:

```text
Wire yamlio into DryRunManifestWriter without adding live execution behavior.
```

This is a planning document only.

It does not change runtime behavior.

## Current Gate

```bash
git status --short
```

Required result:

```text
- go test ./... completes successfully
- go mod tidy does not produce unexpected module metadata changes
- any generated go.sum is committed only if produced by Go tooling
```

Current status:

```text
PR-037 static stabilization complete
real go test ./... pending
CI visibility unknown
```

## Current yamlio Boundary

Current package:

```text
integrations/gitops/yamlio
```

Current functions:

```go
ReadManagedCluster(data []byte) (api.ManagedCluster, error)
WriteManagedCluster(cluster api.ManagedCluster) ([]byte, error)
```

Current implementation:

```text
Dependency-free narrow ManagedCluster YAML-shaped parser/formatter skeleton.
```

Important limitation:

```text
It is not a general YAML parser.
It only supports the current ManagedCluster dry-run example shape.
```

## Intended Wiring Scope

The first wiring step should keep the existing dry-run contract:

```text
ManifestPatchPlan + ManagedCluster bytes
  -> parse ManagedCluster
  -> ApplyManagedClusterPatch
  -> serialize updated ManagedCluster
  -> return WriteResult / branch-ready metadata
```

Allowed behavior:

```text
- parse provided bytes
- validate ManagedCluster through infra/api
- apply allowlisted replica patch
- serialize deterministic output bytes
- produce dry-run WriteResult metadata
```

Forbidden behavior:

```text
- no filesystem reads
- no filesystem writes
- no GitHub API calls
- no GitHub branch creation
- no pull request creation
- no Kubernetes API calls
- no kubectl apply
- no secret resolution
- no raw credential material
```

## Proposed API Shape

Avoid forcing `DryRunManifestWriter` to read files directly.

Prefer a byte-oriented method first:

```go
func (w *DryRunManifestWriter) WriteManagedClusterBytes(
    plan ManifestPatchPlan,
    input []byte,
) (*ManifestWriteResult, error)
```

Possible flow:

```text
ReadManagedCluster(input)
ApplyManagedClusterPatch(plan, cluster)
WriteManagedCluster(updated)
return result with updated bytes and metadata
```

If `WriteResult` currently does not carry bytes, add a separate field only after checking existing consumers.

Avoid breaking the existing method:

```go
WriteManagedCluster(plan ManifestPatchPlan, cluster api.ManagedCluster)
```

The existing object-oriented method can remain as the lower-level deterministic patch path.

## Test Plan

Add tests that prove:

```text
1. valid ManagedCluster YAML-shaped bytes produce updated output bytes
2. output bytes can be parsed back through yamlio
3. replica change is applied deterministically
4. unsupported field fails closed
5. current value mismatch fails closed
6. invalid input bytes fail closed
7. no file path is read or written
```

Recommended test files:

```text
integrations/gitops/dry_run_writer_yamlio_test.go
```

## Error Mapping

Preserve existing error boundaries where possible:

```text
yamlio.EmptyInput -> GitOpsError or propagated wrapped error
yamlio.InvalidYAML -> parse failure
yamlio.InvalidManagedCluster -> validation failure
ApplyManagedClusterPatch errors -> existing GitOpsError path
```

Do not collapse all errors into a generic failure.

## Dependency Policy

Do not reintroduce `gopkg.in/yaml.v3` in PR-038 unless:

```text
- go mod tidy can be run successfully
- go.sum is generated by Go tooling
- go test ./... passes with the dependency present
```

Until then, use the current dependency-free yamlio skeleton.

## Done Definition

PR-038 is complete only when:

```text
- PR-037 has a real passing test run
- yamlio is wired into a dry-run byte-oriented path
- existing object-oriented DryRunManifestWriter behavior remains compatible
- new yamlio wiring tests pass
- no live execution behavior is introduced
```
